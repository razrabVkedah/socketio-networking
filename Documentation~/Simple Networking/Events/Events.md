Клиент - это singleton. За счёт этого можно обращаться к клиенту через статические методы. В том числе - подписываться на события, вызываемые клиентом.

# Как устроены подписки на события в BestSocketIo?

Клиент может подписаться на событие сколько угодно раз. Но нельзя отписаться от события только одним из обработчиков - отписка от события происходит сразу для всех обработчиков, подписанных на данное событие.

BestSocketIo события имеют еще одну тонкость. Если подписавшийся на событие обработчик ожидает получить int, но в событие поступают данные типа string (которые не удаётся привести к типу int), то на клиент происходит ошибка. Такое происходит, потому что BestSocketIo, несмотря на различие типов, пытается привести данные к нужному виду. Ввиду такой особенности получается, что нельзя подписываться на одно событие обработчиками, ожидающими на вход разные типы данных, потому что код корректно отработает лишь для обработчиков одного из типов данных.

Но при этом можно подписаться на событие, которое не возвращает никаких данных. Тогда может существовать две группы обработчиков для одного события: обработчики не принимающие никаких данных и обработчики принимающие какой-то определённый (одинаковый) тип данных.
